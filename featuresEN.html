User Roles

User Roles

Each organization has a group of Managements and under each group, a group of users. El Archive provides different user roles; each user has one or more roles within the system:

  • System administrator: responsible for system data lists, managing user lists, departments and user permissions.
  • System User: Users can use all system features except the system administrator functions.

Search

Search for documents

The user can search the search settings on a particular document by the fields that are specified after the analysis phase and may include:

  • Document type
  • Document name
  • Document history

Searching in documents

The user can search the documents resulting from the "Search for documents" and can know the number of papers in the document and their order User can:

  • View document data
  • Modify document data
  • View the document

Search for a folder

User can search for a specific folder by the fields specified after the analysis phase and may include:

  • File type
  • File name
  • File history
  • File number

Searching in folders

User can search the documents resulting from "Searching for Documents" and can see the number of documents within a file, the number of pages and the file number

User can:

  • View file data
  • Modify the file
  • View the documents inside the file
  • Browse documents from within files

Project management

The project management consists of several units that give the system administrator many powers and it consists of the following units:

User Management Module

A module that allows an administrator to manage system users and their permissions on the system by:

  • Adding a new user
  • Editing user data
  • Deleting and restore a user
  • Identifying groups to which the user belongs
  • User Permissions

Groups

The system administrator can manage groups in the system by:

  • Add a new system group
  • Modify group data
  • Delete and restore a group
  • Give permissions to a group

Document Type Unit

The system administrator or the authorized person controls the types of documents by:

  • Add document type
  • Modify document type
  • Delete document type
  • Add, edit and delete document type properties
  • Choose the file types that will appear for this administration

Characteristics of document types

The system administrator or the authorized person controls the properties of document types by:

  • The user can choose the property display name
  • The user can add a permanent default value
  • The user can specify the size of the field
  • The user can add field verification
  • The user can arrange the property
  • The user can hide the property

File type module

The system administrator or the authorized administrator can also manage file types via

  • Add a new file type
  • Modify file type data
  • Delete a file type

Properties of file types

The system administrator or the authorized person controls the properties of the file types by:

  • The user can choose the property display name
  • The user can add a permanent default value
  • The user can specify the size of the field
  • The user can add field verification
  • The user can arrange the property
  • The user can hide the feature

Files

Files

The system provides a file management unit that consists of many modules that give users many functions, which is to add documents and files and add documents inside files and control their properties:

Adding a file

Users can enter files and documents to create files in the system and can review and modify documents; the screen also displays the files previously created on the system and the date of their creation and the creator of the file You can also view the number of documents within each file and the number of pages The system allows the display of documents within each file by pressing the "Documents" button and adding new documents in the file can also be modified

When you create a new file, users can scan files and add data to search for

Add a document to the file

To create documents, the data entry operators choose to add a document to the file, from which the scanned files are displayed for the user to divide according to the client's desire. To facilitate the search process for the user.

The user can download the document data via the "Fetch Previous Data" button (the button loads the last data added to the transaction of the same document type) The system shows the file scanned by the scanner and the user fills the agreed index fields after selecting the document type which helps search for documents in the system after entering them.

Modifying documents

If there is an error in the archived documents, data entry can be modified

Reports

Reports

The system provides reporting lists for archiving and documents within the system based on a strong reporting platform to enhance the efficiency of the institution, including reports such as the number of documents and files within the system and measuring archival rates ... etc.

The reports that the client wishes to see through the system are agreed upon with the client.

File Tree

File Tree

The system provides the users the ability to create a file tree to make archiving easier within the system, the folders created are added to the search function to help the user find files inside, the user can add several sub levels for each folder, to make searches specific, and to categorize files.

Users can give each of these folders their own attributes to supplement the search feature and help the users reach their files faster.
